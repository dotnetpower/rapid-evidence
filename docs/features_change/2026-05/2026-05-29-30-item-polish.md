Single commit grouping 30 small UX / backend / docs improvements found
in a walkthrough of every dashboard page on `main` (no separate
branch).

## 1. Backend hygiene + safety

| # | Change |
|---|--------|
| 1 | `ruff --fix` cleaned 5 unused-import F401s (`cli.py`, `orchestrator/scheduler.py`). |
| 2 | New `_no_store_for_realtime` middleware sets `Cache-Control: no-store` on every JSON endpoint that the dashboard polls (`/dashboard`, `/batches`, `/metrics`, `/quota`, `/regions`, `/scaling`, `/jobs`, `/events`, `/pool`, `/health`). Stops corporate proxies / browser disk cache from showing 30-second-old snapshots. |
| 4 | `/regions/status` now includes `as_of` ISO timestamp so callers know the snapshot age. |
| 25 | `BackgroundJobRegistry._snapshot` switched from `dict(...)` (shallow) to `copy.deepcopy` so callers cannot mutate nested probe lists / metadata that the registry still holds. |
| 26 | `QuotaProbeRequest` now has Pydantic `field_validator`s for `requested_per_region` (1..10000), `max_parallelism` (1..64), `per_region_timeout_seconds` (0..600). Invalid payloads return `400` synchronously instead of a 500 / failed job. |
