# 2026-05-30 — Premium 120-item improvement plan

User asked for "최소 메뉴별 20가지 개선사항 더 적용해" (≥20 improvements per
menu, premium quality). This is the master plan covering all 6 sidebar
menus (Throughput, Regions, Batches, Scaling, Quota, Audit) **plus**
cross-cutting AppShell + lib work, **120 items total**.

The codebase invariants (`.github/copilot-instructions.md` §6 hardening
loop, §7 SRP/300-LOC ceiling, §8 step-back-first) mean this **cannot
ship in one turn at premium quality** — 120 items × (design + impl +
test + hardening) is multi-day work.

## Delivery strategy

**Phase 1 (this turn, ~30 items)** — high-impact, low-coupling polish
across every menu. Ships with a hardening cycle. Establishes shared
infrastructure (keyboard hook, toast system, sparkline component, CSV
util) that Phase 2+3 reuse so per-feature LOC stays low.

**Phase 2 (follow-up turn, ~50 items)** — menu-specific deep features
that need new components or backend touch points (e.g. bulk-probe
dialog, command palette, replay mode).

**Phase 3 (final turn, ~40 items)** — backend-coupled features and
advanced analytics (historical quota chart, anomaly detection on
throughput, batch resume/restart, etc).
